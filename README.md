# Pilha
Demonstrando a implementação do conceito de pilha. A pilha pode ser entendida como uma lista ligada um pouco mais complicada ou completa, ela trabalh com os mesmos conceitos de ponteiros, alocação de memória dinâmica, nós, o uso de malloc e free, a diferença é que ela tem uma função para adicionar itens e uma para remover, uma lógica parecida com a das filas, mas um pouco mais simples. O algoritmo começa com a criação da struct do nó, que vai receber um valor inteiro e um ponteiro para apontar para o próximo item, e depois vamos para as funções, normalmente faz-se primeiro a função de adicionar (que na pilha se chama "push"), e depois a de remover (que se chama "pop"). A função push recebe como parâmetros o topo da pilha e um valor inteiro para adicionar, note que é mais simples que a fila porque não precisa do início e do fim como a fila precisa, só o topo basta, a gente começa a função escrevendo a struct do nó novo, fazemos um casting de struct nó para o malloc "saber" que estamos trabalhando com nós e aí declaramos que o espaço reservado na memória deve ser para um struct nó. Feito isso, estabelecemos que o valor do nó vai receber num e que o ponteiro próximo vai apontar para o topo, enquanto o topo da pilha por sua vez irá receber o nó novo, que é o valor armazenado e o endereço. Depois vamos para a função pop para remover itens, ela só recebe o topo como parâmetro, não precisa de um valor inteiro porque ela não vai adicionar nada, dentro dela nós usamos um if/else para verificar se a pilha está vazia, pois se estiver não faz sentido tentar remover um item dela, mas se não estiver nós criamos um outro nó que chamamos de auxiliar, este nó vai receber o endereço do topo da pilha para armazená-lo temporariamente e impedir que a gente perca essa referência antes da hora, senão toda a estrutura da pilha se desfaz, então declaramos que o topo da pilha vai apontar para o próximo e receber um novo valor, e aí liberamos a memória que estávamos reservando para a variável auxiliar, não precisamos mais dessa informação, podemos finalmente descartar o que a variável auxiliar estava guardando, que era o antigo endereço do topo. Feito isso, chegamos na main, onde a struct do topo da pilha começa nula porque está vazia e podemos adicionar itens à fila com push e remover com pop, para imprimir usamos a estrutura "%d\n, topo->valor" se quisermos impimir o que está no topo, se quisermos imprimir o que está embaixo do topo usamos "%d\n, topo->proximo->valor", se por acaso tentarmos remover mais itens do que a pilha possui, será impressa uma imagem avisando que a pilha está vazia.
